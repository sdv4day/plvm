# PLVM: 轻量可嵌入 D 语法脚本引擎 — 项目规划书

**版本**: v2.5  
**状态**: 独立 D 语言库  
**宗旨**: 提供支持 D 语言 BetterC 子集的解释型虚拟机，模板化注册宿主函数、结构体和枚举。脚本执行完毕自动卸载，零外部依赖，无内存操作，完全内存安全。

---

## 1. 项目定位

PLVM 是一个纯 D 实现的解释型脚本引擎，遵循 DUB 包标准。脚本采用 D 语言的 BetterC 子集，支持所有基础类型以及宿主注册的函数、结构体和枚举。通过模板机制极大简化集成，让宿主程序以最少代码暴露自身能力给脚本层。所有系统操作由宿主显式提供，脚本环境完全可控。脚本中无指针操作、无手动内存管理，确保内存安全。

---

## 2. 核心设计原则

- **纯 D 实现**：无 C 依赖，仅依赖 D 标准库。
- **BetterC 子集**：脚本采用 D 语言的 BetterC 子集，无运行时依赖，无 GC，无内存操作。
- **D 语法子集**：脚本用标准 D 书写格式，无需学习新语言。
- **解释执行**：自定义字节码 VM，无需外部编译器。
- **纤程机制（可选，暂不实现）**：基于 D 标准库 `Fiber` 实现协程支持，提供轻量级多任务能力。
- **即用即抛**：`callOnce` 加载脚本 → 调用函数 → 自动卸载。
- **模板化注册**：利用 `__traits` 和 `std.traits` 自动推导函数签名、结构体字段、枚举成员，最小化集成工作量。优先使用 `std.traits` 的内置模板或函数，避免不必要的自行包装。
- **标准库优先**：充分使用 D 语言标准库中的内置功能，包括 `std.sumtype`（和类型）、`std.typecons`（类型构造器）、`std.functional`、`std.variant`（变体类型）等，减少自定义实现。
- **沙箱安全**：限制步数与内存，所有 IO 能力由宿主显式注册。
- **零隐式能力**：脚本无法直接访问文件、网络或系统调用。
- **内存安全**：脚本中无指针操作、无内存分配/释放、无手动内存管理。所有内存操作由 VM 实现，依赖 `IAllocator` 或 `ISharedAllocator` 接口。

## 3. 脚本语言支持

### 3.1 基础类型

完整支持：`void`, `bool`, `byte`, `ubyte`, `short`, `ushort`, `int`, `uint`, `long`, `ulong`, `char`, `wchar`, `dchar`, `string`, `wstring`, `dstring`。

**不支持浮点类型**：`float`, `double`, `real` 等浮点类型不在脚本中支持。

### 3.2 BetterC 子集特性

脚本引擎实现 D 语言的 BetterC 子集，具有以下特性：

- **语法子集**：脚本语法遵循 D 语言的 BetterC 子集，禁止使用 `new`、`delete`、`TypeInfo`、异常等依赖 DRuntime 的特性。
- **脚本侧无原生内存操作**：脚本不能直接分配或释放内存，所有对内存的操作（数组、字符串、闭包等）由 VM 实现。
- **内存管理透明**：VM 通过可插拔分配器（默认 GC）管理内存，脚本本身不会主动触发 GC，但 GC 可能会因为内存压力自动触发停顿。
- **可预测的实时行为**：如果需要可预测的实时行为，宿主应注入非 GC 分配器（如 `MallocAllocator`），避免 GC 停顿。
- **内存安全保证**：无指针运算、无 `@system`/`@trusted` 代码、无悬挂引用。

**虚拟机内置支持**（不依赖外部运行时）：
- **数组支持**：动态数组的创建、访问、切片、追加等操作
- **字符串支持**：`string`、`wstring`、`dstring` 类型的完整支持
- **闭包支持**：局部函数和委托的捕获上下文支持
- **局部变量**：支持函数内的局部变量声明和使用

### 3.3 数组类型

- **动态数组**：`int[]` 等，由 VM 自动管理存储。
- **数组操作**：`.length`、索引、`~=` 追加、切片等均由 VM 执行边界检查和内存分配（通过分配器）。
- **边界检查**：所有数组访问都进行运行时边界检查，确保索引在有效范围内，防止越界访问。
- **性能考虑**：边界检查会带来运行时开销。VM 可在编译时优化已知安全的索引操作，或在特定配置下禁用边界检查（需谨慎使用）。

**字符串类型**：
- `string`：不可变的 UTF-8 字符数组（`immutable(char)[]`）
- `wstring`：不可变的 UTF-16 字符数组（`immutable(wchar)[]`）
- `dstring`：不可变的 UTF-32 字符数组（`immutable(dchar)[]`）
- 字符串字面量支持（`"hello"`、`"world"w`、`"universe"d`）
- 字符串连接操作（`~` 操作符）
- 字符串同样执行边界检查，确保内存安全

### 3.4 复合类型

- **结构体**：宿主通过 `registerStruct` 注册，脚本可声明变量并使用点运算符访问字段。
- **枚举常量**：宿主通过 `registerEnum` 注册，脚本可按 `EnumName.Member` 形式引用。

### 3.5 函数类型

- **嵌套函数**：脚本支持在函数内部声明嵌套函数
- **function 类型**：支持函数指针类型声明
- **delegate 类型**：支持包含上下文的委托类型，用于局部函数调用
- **函数字面量**：支持匿名函数和 lambda 表达式
- **局部函数调用**：支持脚本中的局部函数调用，包括捕获外部变量
- **作用域限制**：脚本内声明的函数、function 和 delegate 仅在脚本作用域内可见，不暴露给宿主

### 3.6 语句与控制流

- 变量声明与赋值（支持 `scope` 域内声明）
- `if`/`else`
- `while` 循环
- `do`-`while` 循环
- `for` 循环
- `foreach` 循环（支持数组遍历）
- `switch` 语句（支持基础类型的分支判断）
- 函数定义与递归调用（受步数限制）
- `return` 语句
- `yield` 语句（可选，纤程让出控制权，暂不实现）
- `cast` 类型转换（由 VM 提供实现）
- 单行注释 (`//`) 与块注释 (`/* */`)

### 3.7 类型转换

脚本支持 `cast` 关键字进行类型转换：

- **语法**：`cast(TargetType)expression`
- **实现方式**：类型转换由 VM 提供实现，确保类型安全
- **支持类型**：
  - 整数类型之间的转换（如 `int` → `long`）
  - 字符类型之间的转换（如 `char` → `wchar`）
  - 枚举类型与基础类型之间的转换
- **安全检查**：VM 在运行时检查转换的合法性，非法转换将抛出异常

### 3.8 未支持

- **指针与内存操作**：
  - 指针类型和指针运算
  - `new`、`delete` 操作符
  - `malloc`、`free` 等内存分配函数
  - `@system` 和 `@trusted` 代码
  - `unsafe` 块
- **面向对象特性**：类、接口、模板、混入
- **异常处理**
- **模块导入**（所有代码在单源文件中）
- **运行时反射**
- **GC 相关操作**

## 4. 虚拟机架构

### 4.1 核心组件

- 64 个通用 64 位寄存器
- 操作栈（可配置大小，默认 1024 槽）
- 局部变量帧（按函数调用分配）
- 内建步数计数器
- 纤程管理器（可选，基于 `core.thread.Fiber`，暂不实现）

### 4.2 指令集

变长编码，涵盖：
- 数据移动（寄存器/立即数/内存）
- 整数算术与逻辑
- 比较与条件分支
- 内部函数调用与宿主函数调用
- 栈操作
- 类型转换
- 纤程控制（可选，yield、resume，暂不实现）

### 4.3 内存模型

结构体和枚举值通过基址 + 编译时偏移访问，由编译器利用宿主注册的布局信息生成高效的 `lw/sw` 序列。

### 4.4 标准库使用

**类型系统实现**：
- **`std.sumtype`**：用于实现脚本值的类型安全表示，支持多种类型的和类型（Sum Type）。
- **`std.variant`**：用于实现动态类型的值存储，支持运行时类型检查和访问。
- **`std.typecons`**：用于实现类型构造器，如 `Nullable!T`、`RefCounted!T` 等。

**具体应用场景**：
- **脚本值表示**：使用 `std.sumtype.SumType!` 或 `std.variant.VariantN` 存储脚本中的动态值。
- **类型安全**：使用 `std.sumtype` 的模式匹配确保类型安全，避免运行时错误。
- **可选值**：使用 `std.typecons.Nullable!T` 表示可能不存在的值。
- **引用计数**：使用 `std.typecons.RefCounted!T` 管理共享资源。

**优势**：
- 减少自定义代码，降低维护成本。
- 利用标准库的优化和 bug 修复。
- 提高代码的可读性和可维护性。

### 4.5 纤程机制（可选，暂不实现）

- **协程支持**：每个纤程拥有独立的执行上下文和栈空间
- **调度方式**：协作式调度，脚本主动让出控制权
- **状态管理**：支持纤程的创建、暂停、恢复和终止
- **内存隔离**：纤程间内存隔离，通过消息传递通信
- **性能优势**：轻量级上下文切换，无需内核参与

**说明**：纤程机制为可选特性，当前版本暂不实现，后续根据需求再行开发。

### 4.6 安全配置选项

**边界检查配置**：
- **默认启用**：所有数组访问默认执行边界检查，确保内存安全。
- **编译时优化**：编译器可优化已知安全的索引操作，消除不必要的边界检查。
- **运行时禁用**：VM 提供配置选项允许在性能关键场景中禁用边界检查（需谨慎使用，可能导致未定义行为）。
- **调试模式**：在调试模式下，边界检查始终启用，并提供详细的错误信息。

**其他安全配置**：
- **步数限制**：可配置最大执行步数，防止无限循环。
- **内存限制**：可配置最大内存使用量，防止内存耗尽。
- **超时控制**：可配置脚本执行超时时间。

## 5. 内存管理

PLVM 虚拟机负责管理脚本执行过程中所有动态内存的分配与释放，脚本层不直接参与任何内存操作。

### 5.1 分配器抽象

- VM 内部通过 `IAllocator` 或 `ISharedAllocator` 接口进行内存分配/释放。
- 默认实现使用 D 运行时提供的 **GC 分配器**（`GCAllocator`），也可在构造 VM 时注入自定义分配器（如 `MallocAllocator`、池化分配器等）。

### 5.2 由 VM 管理的动态内存包括

- 动态数组的存储区（包括扩容时的新存储）
- 字符串连接产生的新字符串
- 闭包环境（嵌套函数捕获的变量）
- 纤程独立的栈空间（可选，若不由 Fiber 自身管理，暂不实现）

### 5.3 数组扩容策略

- **扩容触发**：当数组使用 `~=` 追加元素时，若当前容量不足，VM 自动扩容。
- **扩容算法**：采用指数扩容策略，通常为当前容量的 1.5 倍或 2 倍，以平衡内存使用和性能。
- **内存分配**：扩容时通过分配器分配新的存储空间，并将原数组内容复制到新空间。
- **容量预分配**：VM 可提供 API 允许宿主或脚本预分配数组容量，减少频繁扩容的开销。
- **性能优化**：对于大量追加操作，VM 可优化扩容策略，减少内存分配和复制的次数。

### 5.4 生命周期策略

- **作用域绑定**：数组、闭包环境等动态对象与某个作用域（函数调用、模块）绑定，离开作用域时由 VM 自动释放。
- **无引用计数**：在默认 GC 分配器下依赖 D 的垃圾回收；使用非 GC 分配器时由 VM 在模块卸载或作用域结束时显式释放（需要分配器支持 `deallocate`）。
- **脚本无法显式调用 `delete` 或 `free`**，所有内存回收由 VM 在合适的时机触发。

**闭包的生命周期管理**：
- **捕获机制**：闭包捕获的变量存储在堆上，由 VM 通过分配器管理。
- **生命周期延长**：当闭包被返回或存储到更外层作用域时，捕获的变量生命周期自动延长至闭包的生命周期结束。
- **无悬挂引用**：VM 确保闭包捕获的变量在闭包存活期间不会被释放，避免悬挂引用问题。
- **自动回收**：当闭包不再被引用时，VM 通过 GC 或显式释放回收闭包环境占用的内存。

### 5.5 与 BetterC / 无 GC 声明的关系

- 脚本语言本身遵循 D BetterC 语法子集，不依赖 DRuntime 的 GC 分配，但 **VM 实现可以使用 GC**。
- 若需要完全无 GC 的环境（如实时系统），宿主可注入 `MallocAllocator` 并确保所有脚本执行完毕后显式释放模块。此时 VM 仍能满足"脚本不触碰内存管理"的安全承诺。

## 6. 编译流程

1. **词法分析** → Token 流  
2. **语法分析** → AST（递归下降解析器）  
3. **语义分析** → 类型检查、作用域解析、枚举常量折叠  
4. **代码生成** → 字节码 + 常量池 + 函数表 + 结构体布局引用 + 枚举值映射  

编译产物为 `PlvmModule` 对象，可直接执行或缓存复用。

## 7. 生命周期管理

### 7.1 一次性调用

```d
Plvm plvm;
plvm.registerFunction!myHostFunc;
plvm.registerStruct!MyStruct;
plvm.registerEnum!MyEnum;

plvm.callOnce("script.d", "main", arg1, arg2);
```

`callOnce` 自动编译源码、执行指定函数并释放所有资源。

### 7.2 手动管理（复用模块）

```d
ModHandle h = plvm.loadScript("test.d");
plvm.callFunction(h, "init");
plvm.callFunction(h, "process", data);
plvm.unloadScript(h);
```

### 7.3 VM 实例共享全局状态

每个 VM 执行实例共享全局状态，包括：

- **类型表**：所有注册的结构体和枚举类型信息
- **函数表**：所有注册的宿主函数信息
- **常量池**：编译时常量和枚举值
- **模块缓存**：已编译的脚本模块

**优势**：
- 减少内存占用，避免重复存储
- 提高加载速度，复用已编译模块
- 统一管理，确保类型一致性
- 支持多实例并发执行

## 8. 宿主 API 注册

### 8.1 函数注册

```d
// 自动推导函数名 "myFunc" 和参数/返回类型
vm.registerFunction!myFunc;

// 显式指定脚本中使用的名称
vm.registerFunction!myInternalFunc("publicName");
```

### 8.2 结构体注册

```d
// 自动推导结构体名 "MyData" 及所有字段
vm.registerStruct!MyData;

// 自定义名称
vm.registerStruct!InternalData("SharedData");
```

内部使用 `__traits(allMembers, T)` 和 `std.traits.FieldTypeTuple` 遍历字段，无需手动描述布局。优先使用 `std.traits` 提供的标准功能。

### 8.3 枚举注册

```d
// 自动推导枚举名 "Status" 及所有成员
vm.registerEnum!Status;

// 自定义名称
vm.registerEnum!InternalFlags("Flags");
```

模板自动提取枚举基类型和各成员值，脚本中可以 `Status.Active` 形式使用。

### 8.4 常量注册

```d
vm.registerConstant("MaxRetries", 5);
```

### 8.5 沙箱配置

```d
vm.setSandbox(maxSteps: 10_000, maxMem: 1024 * 1024);
```

## 9. 枚举注册的实现要点

- 使用 `std.traits.isNumeric!T` 和 `__traits(allMembers, T)` 静态遍历枚举成员。优先使用 `std.traits` 提供的标准功能。
- 提取每个成员的名称和 `cast(long)` 值（统一使用有符号 64 位，保留符号信息，符合脚本算术运算的有符号语义）。
- 构建映射表，在脚本编译器内将形如 `EnumName.Member` 的 Token 解析为对应常量。
- 若脚本使用的枚举宿主未注册，编译阶段报错。

**技术说明**：
- 选择 `long` 而非 `ulong` 的原因：保留负值的符号信息，避免负值转换为无符号数后产生异常结果。
- 所有整数枚举类型（包括 `byte`、`ubyte`、`short`、`ushort`、`int`、`uint`、`long`、`ulong`）统一存储为 `long`。
- 脚本中枚举值参与算术运算时使用有符号语义，符合开发者直觉。

## 10. 沙箱与安全

- **步数限制**：每条指令递增计数器，超限即抛出 `PlvmException`。
- **内存限制**：局部变量帧 + 操作栈总字节数受最大内存参数控制。
- **纯能力模型**：引擎内核不提供文件、网络、控制台等能力；所有外部交互必须通过宿主注册的函数实现。这使得脚本运行在一个完全由应用定义的能力沙箱中。

---

## 11. 错误处理

**脚本错误处理机制**：
- **禁止异常**：脚本中禁止使用 `throw`、`try`、`catch`、`finally` 等异常处理语句。
- **错误传递方式**：脚本通过返回值传递错误信息，或调用宿主注册的错误处理函数。
- **宿主函数错误**：宿主函数可以返回错误码或特殊值（如 `-1`、`null`），脚本通过检查返回值处理错误。

**VM 错误处理**：
- **编译错误** → `PlvmCompileException`（行号、列、描述）
- **运行时错误** → `PlvmRuntimeException`（非法指令、除零、超过步数/内存限制、类型转换失败等）

**宿主错误处理**：
- 宿主可以在调用函数周围捕获 VM 抛出的异常，按需进行清理或回滚。
- 宿主可以注册自定义错误处理函数，在脚本运行时错误发生时执行特定逻辑。
- VM 提供错误回调机制，允许宿主在错误发生时获取详细的上下文信息（如调用栈、变量状态等）。

## 12. 集成示例

```d
import plvm;

// 宿主侧定义
int multiply(int x, int y) { return x * y; }
struct Vec2 { int x; int y; }
enum Color { Red = 1, Green = 2, Blue = 4 }

void main() {
    Plvm vm;

    // 模板化注册
    vm.registerFunction!multiply;
    vm.registerStruct!Vec2;
    vm.registerEnum!Color;
    vm.registerConstant("Scale", 10);

    vm.callOnce(q{
        Vec2 v;
        v.x = Color.Red * Scale;
        v.y = multiply(3, 4);

        int total = v.x + v.y;
    }, "main");
}
```

## 13. 技术栈

- **语言**：100% D，DUB 包，ldc 编译
- **依赖**：无第三方库，充分使用 D 标准库
- **标准库使用**：
